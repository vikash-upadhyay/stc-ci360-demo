1
End-to-End Demo Workflow
🌐
Web

Visitor lands on Postpaid Plans page (index.html)

The SAS CI 360 tag fires automatically. A page event is sent with pageType: "postpaid_plans". CI 360 identifies the visitor (anonymous or known) and evaluates which creative to show in the hero banner spot.

CI 360 event → ci360('send', 'page', { pageType: 'postpaid_plans' })
⚡
CI 360

CI 360 serves personalized creative into hero banner spot

Based on visitor segment (new, returning, high-value look-alike), CI 360 injects the appropriate creative HTML into #ci360-hero-spot (Zone: postpaid_hero_banner). Example: a flash offer for annual billing, or a loyalty reward for returning visitors.

Zone: postpaid_hero_banner · Task type: Web Personalization
🛒
Web

Visitor clicks "Add to Cart" on a plan

An addToCart event is fired with the plan name, tier, and price. This enriches the visitor's CI 360 profile in real time, which feeds the upsell decision engine.

CI 360 event → ci360('send', 'event', { eventType: 'addToCart', planName: 'Go 150', planPrice: 199 })
📋
Web

Visitor reaches Checkout page (checkout.html)

The page event fires with pageType: "checkout" and cart data. Then a checkoutReached event is fired with the cart value and plan tier — triggering the CI 360 upsell decision.

CI 360 event → ci360('send', 'event', { eventType: 'checkoutReached', cartValue: 199, planTier: 'Popular' })
🎯
CI 360

CI 360 serves targeted upsell in checkout spot

CI 360 evaluates the cart contents and visitor segment to select the best upsell offer. A Go 150 buyer might see a device bundle; a Go 300 buyer might see roaming add-on. The creative is injected into #ci360-upsell-spot (Zone: checkout_upsell).

Zone: checkout_upsell · Decision: plan-tier-based offer selection
⬆️
Web

Visitor accepts (or dismisses) the upsell

Both outcomes send a CI 360 event: upsellAccepted or upsellDismissed. Acceptance updates the summary and order total. This event feeds back into CI 360 for attribution and future targeting.

Events: upsellAccepted / upsellDismissed · Zone: checkout_upsell
✅
CI 360

Order confirmed → purchase event sent

A purchase event is fired with order ID, total value, plan list, and upsell status. CI 360 records this as a conversion. The data flows into CI 360 Explore (analytics) and feeds suppression lists, loyalty segments, and retention tasks.

CI 360 event → ci360('send', 'event', { eventType: 'purchase', orderId: '...', totalValue: 348 })

upsellImpression checkout.html Upsell creative shown offerType, offerName, zone Impression tracking, frequency cap
upsellAccepted checkout.html Accept upsell click offerName, upsellValue Conversion goal, revenue attribution
upsellDismissed checkout.html Dismiss click zone Suppression (don't re-show same offer)
purchase checkout.html Confirm order click orderId, totalValue, plans, upsellAccepted Revenue reporting, retention suppression

Suggested Audience Segments
🆕 New Visitor — Acquisition

Target: first-time visitors, no prior CI 360 cookie

Show: Starter plan highlight, sign-up incentive (e.g. first month free)

No prior visit Source: Organic / Paid
🔄 Returning Visitor — Nurture

Target: visited plans page 2+ times in last 14 days, no purchase

Show: Urgency flash offer, comparison tool prompt

Visit count ≥ 2 No purchase
⭐ High-Intent — Popular Plan Viewer

Target: spent 30+ seconds on Go 150 or higher plans

Show: Social proof banner ("10,000+ subscribers this week")

Time on plan ≥ 30s Plan tier: Popular+
💎 Premium Prospect — Upsell Target

Target: added Go 150+ to cart, reached checkout

Show: Device bundle upsell or annual billing discount

addToCart: Go 150+ checkoutReached

Deploy to GitHub Pages
# 1. Create a new GitHub repository
gh repo create stc-ci360-demo --public --clone
cd stc-ci360-demo

# 2. Copy your HTML files
cp index.html checkout.html creative.html ci360-guide.html .

# 3. Push to main branch
git add .
git commit -m "Add STC CI360 postpaid demo"
git push origin main

# 4. Enable GitHub Pages
#    Settings → Pages → Source: Deploy from branch → main → / (root)
#    Your site: https://<username>.github.io/stc-ci360-demo/
✅ Once deployed, register the GitHub Pages URL as an allowed domain in your CI 360 tenant settings under Administration → Web → Allowed Domains so the tag can fire correctly on your demo site.

Demo Tips for the Client Presentation
  • 🎯
    Open two browsers — use Chrome (anonymous visitor) and a profile browser (returning visitor) to show different creatives served by CI 360 to different segments simultaneously.
  • 🔍
    Show the CI 360 Explore dashboard live — as you add to cart and proceed to checkout on the demo site, the page events and activity should appear in near real-time in CI 360's visitor journey view.
  • 📊
    Pre-load the analytics — set up a Goal report in CI 360 beforehand tracking upsellAccepted so clients can see conversion lift projected from the demo interactions.
  • 💡
    Narrate the data flow — open DevTools Console to show the [CI360] log messages being sent in real-time as you navigate, demonstrating the live data pipeline without needing backend access.
  • 🔁
    A/B test story — explain how Task 2 (checkout upsell) can be split between a device bundle offer vs. a roaming add-on, with CI 360 auto-optimising traffic to the winner after statistical significance.
